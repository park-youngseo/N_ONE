## 🔹 **내부 필드 정리 (형이 안 골라도 자동으로 들어갈 것들)** 

형이 선택한 4개 말고, 

**크롤러가 제대로 돌려면 최소한 추가로 필요한 내부 정보** 들이 있어: 

- `id` : DB 안에서 쓰는 번호 (1,2,3… 자동 증가) 

- `url` : 글 주소 

- `crawled_at` : 언제 긁었는지(YYYY-MM-DD HH:MM) 

- `hash` : 제목+본문을 섞어서 만든 “내용 지문” 같은 값 → 중복 체크용 

그래서 **게시글 1개** 는 결국 이런 식으로 저장될 거야: 

- id 

- url 

- 제목 

- 본문 

- 좋아요 수 

- hash (내용 지문) 

- crawled_at (언제 긁었는지) 

형이 Q7에서 고른 4개는 “분석용 필드”고, 

나머지는 “크롤러가 똑똑해지려고 몰래 들고 있는 필드”라고 보면 된다.
