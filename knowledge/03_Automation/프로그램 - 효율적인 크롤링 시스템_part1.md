## **① 전체 개요(A단계) – “효율적인 크롤링 시스템”의 뼈대** 

## **효율적인 시스템 =** 

“코드 1개 잘 만드는 것”이 아니라 

**(1) 수집할 것 정의 → (2) 구조 설계 → (3) 자동화·로그 → (4) 예외 처리** 이 4가지만 명확하면 돼.

예:

- 유튜브/블로그의 **배우 지망생 고민 글, 댓글, 제목/태그** 

- 공모, 오디션, 촬영 관련 **공지 페이지** 

- → “엔오원 운영·마케팅에 직접 쓸 데이터”만 목표로 잡는 게 효율적. 

## **2. 시스템 구조 4단계** 

**1. 수집 모듈** : 웹페이지 가져오기 (requests, playwright/selenium) 

**2. 파싱 모듈** : HTML에서 필요한 정보만 뽑기 (BeautifulSoup, XPath 등) 

**3. 저장 모듈** : CSV / 엑셀 / DB / 노션 API로 정리 

**4. 스케줄러** : 정해진 시간마다 자동 실행 (윈도우 작업 스케줄러 or cron) 

## **3. 필수로 넣어야 할 3가지** 

- **로그** : 언제 어디서 에러 났는지 텍스트로 기록 

- **반복 가능한 구조** : 새 사이트 추가할 때 “함수 하나만 더 만들면 되는 구조”

## **1. 요청 사이 딜레이** 

`time.sleep(1~3초)` 정도 랜덤으로 

너무 빠르게 요청 보내면 429(Too Many Requests) 잘 뜸. 

## **2. User-Agent 변경** 

헤더에 진짜 브라우저처럼: 

```
"User-Agent": "Mozilla/5.0 ..."
```

필요하면 여러 개를 만들어서 랜덤으로 돌릴 수 있음. 

## **3. 재시도 로직** 

500번대 에러 or 429 나오면: 

로그 찍고
