깊이 2단계는 뭐야? 최대 200페이지를 읽는 방식은 어떻 게 진행되고? 

## 형, 아주 쉽게 풀어줄게. 

## **깊이 2단계가 뭐야?** 

**0단계** : 내가 고른 첫 페이지(시드). 

- **1단계** : 0단계 페이지 안의 링크들(같은 사이트만). 

- **2단계** : 1단계 페이지 안의 링크들(역시 같은 사이트만). 

- → 여기까지만 들어가고 **3단계는 안 감** . 

## **200페이지는 어떻게 세?** 

- **큐(줄서기)**를 만들어. 

- 0단계 → 1단계 → 2단계 순서대로 **차례차례** 방문해. 

- 방문할 때마다 “읽은 페이지 수+1”. 

- **200** 이 되면 멈춰. 

- 200개가 안 되면, 더 갈 링크가 없을 때 멈춰. 

## **실제 흐름(간단)** 

**1.** 시드 URL 정함(선택한 도메인의 첫 링크). 

**2. 같은 도메인** 의 링크만 수집(외부, mailto, javascript 제외). 

**3.** 이미 본 링크는 **중복 건너뜀** . 

**4.** 요청 사이에 **0.5~0.6초** 쉬어줌(예의). 

**5.** 페이지 열리면 

제목, 메타설명, 본문(읽기 좋은 버전) 뽑음. 

문장 나눠서 **질문/고민 문장** 만 골라 저장. 

**6.** 2단계까지 돌거나, 200페이지 찰 때까지 반복. 

## **왜 이렇게 해?** 

- 너무 깊이 가면 **시간 오래** 걸리고 **쓸모 적은 하위 페이지** 가 많아져. 2단계면 “사이트의 주요 섹션”까지는 넓게 보되, **잡음은 줄임** .
