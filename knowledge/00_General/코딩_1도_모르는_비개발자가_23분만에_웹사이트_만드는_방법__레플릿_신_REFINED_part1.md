## [원본 내용 — 무손실 전문 (Part 1)]

> **[AX 하네스 준수]** 아래는 원본 자막 텍스트의 무손실 전문입니다.
> 요약 또는 압축 없이 원자 단위로 전체 내용을 보존합니다.

```
바이브 코딩을 할 때 초보자들이 가장
많이 빠지는 함정이 하나 있습니다.
바로 단순하게 코딩을 통해서 뭔가
결과가 만들어진다고 해서 이게 완성이
된다라고 착각하는 건데요. 겉으로
보기에는 뭔가 멀쩡이 작동을 하는 것
갖다가도 그 실상을 파악하면 사실
에러가 나곤 합니다. 이게 사실
바이브 코딩을 통한 결과물들이 제대로
작동하기 위해서는 내부적으로 필요한
요소들이 정확하게 연결이 되어 있어야
합니다. 이게 대표적으로는 API
연결이 있는데 초보자분들에게는 어려울
수도 있습니다. 그래서 오늘
영상에서는이 부분을 훨씬 더 쉽고
간편하게 하는 방법에 대해서 제가
빠르게 알려 드리도록 하겠습니다.
원래 기존에는 이런 연결하는 부분에
있어서 한도 끝도 없이 복잡해질 수
있었는데 초보자분들은 아주 기겁을
하고 도망을 치시겠죠. 훨씬 더 쉬운
방법이 있습니다. 지금 화면에 보이는
것처럼 구글 드라이브, G, 노션,
유튜브, 디스코드, 기터브 이런
여러분이 자주 사용하시는 앱에
단순하게 로그인 하시면 바로 연결이
끝나 버리는데요. 네. 여러분들께
정말 익숙한 방식일 겁니다. 근데
이게 바이브 코딩에선 아직까지 기능이
없었어요. 근데 이번에 새로운 기능이
나왔습니다. 그래서 바로 알려 드릴
예정이고요. 본론에 앞서서 영상에
좋아요와 채널의 구독 버튼 눌러
주시고 바로 시작해 보도록
하겠습니다. 자, 지금 보시는 화면은
레플릿의 대시보드인데요. 아마 많은
분들이이 레플릿에 대해서 들어보셨을
겁니다. 다이브 코딩 툴 중에서도
되게 신기한 기능들을 많이 출시하는
앱인데요. 지금 왼쪽에 보시면
인티그레이션이라고 새로운 기능이
출시된 걸 확인할 수가 있습니다.
해당 탭을 눌러 보시면 이렇게
인테그레이션이라고 하면서 정말 다양한
앱들을 여러분이 단순하게 로그인을
하는 것으로 이용할 수 있게
되는데요. 오늘 영상에서 보여 드릴
건 간단합니다. 저는 유튜브를
운영하고 있기 때문에 항상 콘텐츠
제작에 있어서 어려움을 겪습니다.
어떤 콘텐츠를 해야지 제 시청자분들이
좋아하실지 또 좋은 반응을 얻을지.
그래서 유튜브 연결을 보시면 내용
중에 채널 통계에 접속을 할 수
있다라고 합니다. 이런 채널 통계
같은 경우는 기존에는 보여 드렸던
것처럼이 구글 클라우드 콘솔을 제가
직접 따와 가지고 연결을 복잡하게
했어야 되는데 이제 단순하게 이제
단순하게 유튜브에 로그인 한다면 바로
사용이 가능하다라는 겁니다. 자,
그래서 우측에 있는 사인 버튼을
클릭을 해 주시고요. 유튜브에 연결을
해서 제 채널의 통계를 쭉 살피게
하고 제가 어떤 콘텐츠를 만들면
좋을지 AI한테 추천을 받는 그런
간단한 앱을 만들어 보도록
하겠습니다. 자, 여기서 사인 버튼을
눌러 주시고요. 그러면 커넥트
유튜브라고 하면서 유튜브로 연결이
될테니까 거기서 로그인을 해 달라라고
합니다. 밑에 컨티뉴투 유튜브 버튼을
클릭을 해 주시면 이렇게 바로 제
채널로 연결이 되고요.이 레플릿에서
어떤 거에 엑세스를 할 수 있을지
항목만 선택을 해 주시면 됩니다.
여기 보시면 제가 정말 다양한
데이터들을 연결을 할 수가 있는데
여러분이 어떤 데이터를 연결하느냐에
따라서 바이브 코딩을 통해서 훨씬 더
좋은 퀄리티의 웹을 만들 수 있을
겁니다. 자, 저는 우선 유튜브
콘텐츠에 대한 유튜브 분석 보고서
조회 이게 가장 필요하기 때문에
여기에 체크를 해 줄 거고요.
여러분은 필요에 따라서 나머지
데이터들을 연결을 시켜 주셔도
됩니다. 스크롤을 밑으로 내려서
계속을 눌러 주도록 합니다. 그러면
지금 커넥션이 완료가 됐다라고 뜨면서
위에 가시면 유튜브가 연결이 완료가
되면서이 커넥션 스터스가 액티브로
바뀐 걸 확인할 수가 있습니다. 자,
그러면 연결이 됐기 때문에 홈으로
가서 제가 한번 웹을 만들어 보도록
하겠습니다. 자, 저는 간단하게
프롬프트를 적어 줬는데요. 제 유튜브
콘텐츠를 분석한 이후에 어떤 콘텐츠를
앞으로 만들면 좋을지이 방향성을
잡아주는 콘텐츠 추천앱을 만들고
싶다라고 했습니다. 앞으로 4주 동안
어떤 콘텐츠들을 몇 번씩 올려야
되는지 이런 부분까지 제가 찾아
달라고 했습니다. 웹을 만들고 싶기
때문에이 하단에 웹을 선택을 해
주고요. 스타트 챗을 눌러서 한번
어떤 웹을 만들어 주는지 확인을 해
보도록 하겠습니다. 현재 플랜 단계를
하고 있죠. 유튜브 콘텐츠 분석을
만들어 주겠다. 그러면서 본인들이
연결되어 있는 인티그레이션을 확인을
하니까 레플릿 인티그레이션 즉
커넥터에 유튜브가 있다는 걸 확인을
할 수 있다라고 합니다.이 하단에
작은 팝법이 뜨면서 유튜브와 연결을
승략하시겠습니까라는
질문이 나옵니다. 저는 필요하기
때문에 컨티뉴 하겠다라고 버튼을
클릭을 해 주고요. 자, 유튜브 연동
승인이 완료가 되었다라고 하면서
콘텐츠 분석 및 추천 앱 계획을
제한하겠다라고 합니다. 자, 그러면
이렇게 전체 플랜이 나오는데 유튜브
API 연동으로 채널의 모든 영상
데이터 자동 수집이라고 하는데 이건
저희가 할 필요가 없어요. 왜냐면
이미 저희는 유튜브를 커넥터로 연결을
시켜 줬기 때문에 알아서 할 겁니다.
이런 부분에서 연결해서 정말
편리하기도 하고 쉽기도 합니다.
그리고 제가 프롬프트에 입력한 거
여기 잘 들어가 있고요. 그다음
마지막으로 사주 콘텐츠 캘린더를
생성을 하겠다. 이후에는 계획표를
만든 다음에 구글 캘린더까지 연동을
시켜서 이것도 훨씬 더 쉬워졌습니다.
제가 직접 받아보도록 하겠습니다.
아, 제가 이걸 조금 더 확대해서
보여 드리면 두 가지 결과물이 있는데
첫 번째 결과물은 레플릿한테 모든 걸
맡기는 겁니다. 이번에 에이전트
기능도 새로 나온 기능 중에 하난데
제가 입력한 프롬프트에 맞춰서
에이전트가 디버깅도 하고 전체적인
앱을 쭉 만들어 주는 겁니다. 20분
이상 걸린다라고 하고요. 우선요
기능은 제가 나중에 따로 설명을
드리도록 하겠습니다. 자, 오른쪽은
우선 프로토타입을 만드는 기능입니다.
그래서 처음 만든 결과물의 디자인이나
기능 이런 부분에서 제가 미세하게
조정하면서 원하는 방향으로 가는 건데
저는이 두 번째 방법을 선택을 하고
한번 어떤 디자인을 먼저 만들어 줄지
먼저 한번 보고 나서 제가 방향성을
조정해 보도록 하겠습니다. 모든
선택이 완료가 됐으면 가장 하단에
있는 버튼을 눌러서 그다음 순서로
이어가도록 하겠습니다. 자, 그러면이
코드 진행이 되고 있는이
워크스페이스를 설명해 주는 팝법이
뜨는데요.이 왼쪽은 플릿
에이전트입니다. 여러분이 여기에
프롬프트를 입력을 하고 코드에 변화를
주는 공간이고요. 우측 상단에 플러스
버튼을 눌러서 여러분이 여러 가지
데이터베이스를 연결한다던가 아까
처음에 보셨던 인티그레이션이 커넥터를
연결을 해서 다양한 서비스를들을
바로바로 가져올 수가 있습니다.
그리고 바이브 코딩을 할 때 사실
보안이 약한 부분이 있는데 래플릭
같은 경우는 보완도 자동으로 연결이
되면서 훨씬 더 편안하게 앱을 만들
수가 있습니다. 그리고 우측에 현재
만들어지는 파일들을 실시간으로 확인을
할 수가 있고요. 마지막으로이 앱이
완성이 됐으면 우측에 퍼블리시를
눌러서 바로 공유를 할 수 있는
장점이 있습니다. 기존에는 코드를
만들어서 내가 따로 배포를 해야
됐다면 레플릿을 사용하면은 그런
걱정을 하지 않으셔도 된다는 장점이
있습니다. 자, 이렇게 대시보드
설명이 끝났고요. 제가 3분 전에
만들어 달라고 했는데 지금 벌써
결과물을 거의 다 만들어 줬거든요.
우측에 보시면 현재 에이전트가
전체적인 디자인을 완성을 시키고
있다라고 하는데 프로토타입이 완성이
되었다라고 하면서 디자인을 확인해
보라고 합니다. 밑에 내려가서이 뷰
프리뷰 버튼 있죠?이 이 버튼을
누르면 예시 데이터가 들어간이
디자인을 확인할 수가 있습니다.
로고도 만들어 줬고요. 대시보드에서
주제별 평균 성과도 이렇게 확인할 수
있다라고 하고요. 어떤 영상이
고성과였는지 이런 부분도 제가 확인을
할 수가 있을 것 같습니다. 그래서
왼쪽에 콘텐츠 계획을 눌러 보면
이틀에 한 번 꼴로 고성능 주제에
관해서 이렇게 컨텐츠를 만들면 좋을
것 같다라고 합니다. 깔끔한 색감도
괜찮고 레이아웃도 괜찮기 때문에 제가
이제는 기능을 만들어 달라고 할
겁니다. 자, 레플릿으로 돌아와서
왼쪽 하단에 보면 이제 기능을 추가해
달라라는 버튼이 있습니다.이 버튼을
누르면 그다음 단계로 진행이 되는데
그 전에 여러분이 한 가지 추가해
주셔야 되는 부분이 있습니다. 아까
저희 앱 계획서를 보면 인티그레이션에
유튜브 당연히 있었죠. 왼쪽에 오픈
AI가 있었습니다. 이게 필요했던
이유는 나중에 저희가 만든 앱이 AI
앱이기 때문에 AI 모델이 필요한
거거든요. 근데 여기서는 채치피pt를
쓸 거라는 겁니다. 레플릿에서
커넥터를 새로 출시하긴 했지만 이렇게
API가 꼭 필요한 부분에서는
여러분이 해당 키를 가져오시기만 하면
됩니다. 자, 그래서 빠르게 오픈
AI로 가서 API 키 만들어 주기
버튼을 눌러 주시고요. 저는 레플릭
키라고 이름을 만든 다음에 바로 키를
만들도록 하겠습니다. 이걸 바로
카피를 해 주시고요. 레플릿으로
돌아와서이 밸류에 붙여 넣기만 해
주시면 모든게 끝입니다. 뭐 더 할게
```
