# [Master 05-02] 자동화 엔진 보안 및 환경 관리 (The Immune System)

사령관님, 엔진이 돌아가는 것보다 중요한 것은 그 엔진이 **'안전하게'** 그리고 **'지속 가능하게'** 돌아가는 것입니다.

## 1. 비밀번호 금고 (Secret Management)
- **위험**: 코드에 `api_key = "sk-..."`라고 적는 순간, 보안 사고의 위험에 노출됩니다.
- **해결**: `.env` 파일을 사용하여 키를 분리하십시오.
    - 파이썬에서 `python-dotenv` 라이브러리를 사용하여 안전하게 불러옵니다.
    - **핵심**: `.gitignore` 파일에 `.env`를 등록하여 절대 외부로 유출되지 않게 차단하십시오.

## 2. 환경 격리 (Environment Isolation)
- **위험**: 여러 프로젝트의 라이브러리 버전이 충돌하면 전체 시스템이 붕괴합니다.
- **해결**: `venv` 또는 `Conda`를 활용하여 각 프로젝트마다 독립된 환경을 구축하십시오.
    - 명령: `python -m venv .venv` -> `.venv\Scripts\activate`
    - **효과**: "내 컴에서는 잘 되는데 왜 서버에서는 안 되지?"라는 문제를 원천 차단합니다.

## 3. 타임머신 복구 (Version Control)
- **위험**: 코드 수정 실수로 어제까지 잘 벌리던 수익이 0원이 되는 상황.
- **해결**: `Git`을 활용하여 매일의 성과를 기록하십시오.
    - **Commit**: 성공적인 수정 시마다 '체크포인트'를 생성합니다.
    - **Rollback**: 문제 발생 시 즉시 가장 안정적이었던 과거 버전으로 복구합니다.

## 🚀 NONE-OS 시스템 방어 수칙
- **규칙 1**: 모든 API 키는 반드시 `.env`를 통한다.
- **규칙 2**: 새로운 자동화 툴 설치 시 반드시 가상 환경을 먼저 만든다.
- **규칙 3**: 하루의 업무가 끝나면 반드시 Git으로 기록(Commit)을 남긴다.
