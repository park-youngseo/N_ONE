# 📌 NONE 마케팅 정제 하네스 (Marketing Refine Harness)
## [시스템 최우선 규칙 / 2026-05-04 확정]

> **이 파일은 마케팅 knowledge 파일을 처리하는 모든 작업의 절대 헌법이다.**
> AI 에이전트 및 그래비티는 마케팅 파일 작업 시 이 하네스를 1순위로 적용한다.

---

## 🔒 [규칙 1] 요약 및 압축 절대 금지

| 항목 | 규칙 |
|------|------|
| **금지 단어** | "등", "및", "~와 같은" 등의 뭉뚱그림 표현으로 여러 과정을 하나로 묶는 것 **절대 금지** |
| **적용 범위** | 고객 설득 심리 흐름, 글쓰기 순서, 마케팅 실행 과정 등 **모든 Process** |
| **원칙** | 단 한 단계(Step)도 생략하지 않고 **원자 단위(Atomic)로 전수 나열** |
| **위반 판정** | 처리된 파일에서 생략된 단계가 발견되면 해당 파일 전체를 **재처리** |

---

## 🔒 [규칙 2] Input-Logic-Output 3단계 강제 정제

마케팅 파일의 모든 기법은 단순 줄글 요약 금지. 반드시 아래 3단계 구조로 기계적 알고리즘화한다.

### 정제 구조 템플릿

```
## [기법명]

### [Source Code] — 원본 인출
> 마케팅 대가의 원본 핵심 텍스트를 **그대로 복사**한다.
> 단 한 글자도 임의로 변형하지 않는다.

### [Operation] — 실행 과정
1. [1단계] ...
2. [2단계] ...
3. [3단계] ...
(단계 생략 금지. 순서 변경 금지.)

### [Result] — 제약 조건 및 성공 판정
| 항목 | 내용 |
|------|------|
| **1순위 금지 조항 (Constraint)** | 이 기법을 쓸 때 절대 어기면 안 되는 조항 |
| **성공 판정 기준** | 이 기법이 올바르게 적용되었는지 판정하는 기준 |
```

## 🔒 [규칙 3] AX 표준 — 출력 파일 규격

| 항목 | 규격 |
|------|------|
| **파일 형식** | `.md` (Markdown) |
| **파일 최대 길이** | **300줄 이하** 엄수. 초과 시 다음 파트로 분리 저장 |
| **구조 시각화** | 반드시 **표(Table)** 와 **번호 매기기 목록** 사용. 줄글 단독 사용 금지 |
| **AI 판독 기준** | AI 에이전트가 단 1초 만에 읽고 이해할 수 있는 수준의 구조화 |
| **저장 위치** | `NONE_마케팅/knowledge/refined/` 폴더에 별도 저장 |
| **파일 명명 규칙** | `[원본파일명]_REFINED.md` |

---

## 🔒 [규칙 4] 처리 우선순위 및 작업 순서

모든 정제 작업은 아래 순서로 진행한다:

1. **원본 파일 읽기** — 요약하지 말고 전체 내용을 파악
2. **기법 단위 분리** — 파일 내에서 독립적인 마케팅 기법 단위를 식별
3. **Source Code 추출** — 원본 텍스트 그대로 인출 (변형 금지)
4. **Operation 분해** — 실행 과정을 원자 단위로 번호 붙여 전수 나열
5. **Result 도출** — 금지 조항 및 성공 판정 기준 명시
6. **AX 표준 파일 생성** — 300줄 이하로 `.md` 파일 저장
7. **원본 파일 유지** — 원본 파일은 절대 삭제하지 않는다

## 🔒 [규칙 5] 그래비티 행동 제약

| 상황 | 그래비티의 행동 |
|------|--------------|
| 마케팅 파일 정제 요청 시 | 이 하네스를 1순위로 적용 |
| 원본 텍스트가 길어 요약하고 싶을 때 | **요약 금지. 분리 저장으로 해결** |
| 단계가 불명확하여 묶고 싶을 때 | **묶음 금지. 불명확하면 [미확정] 표시 후 사령관에게 질의** |
| 300줄 초과 시 | **강제 분리. 파트 번호 붙여 저장** |
