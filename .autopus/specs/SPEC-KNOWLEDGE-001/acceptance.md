# SPEC-KNOWLEDGE-001 Acceptance: 지식 습득 시스템의 품질 검증 로직 설계

## Test Scenarios

### Scenario 1: 무손실 검증 통과 및 자가 증명 출력
- **Given**: 유효한 텍스트 파일(100줄) URL을 입력받은 상태
- **When**: 시스템이 해당 URL의 지식을 섭취할 때
- **Then**: 터미널에 `[무손실 수집 증명] 라인 수: 100`과 `[원본 첫 3줄: ...]` 및 `[원본 끝 3줄: ...]`이 정확히 일치하여 출력되며, 프로세스는 정상적으로 다음 단계로 진행된다.

### Scenario 2: 시맨틱 중복 제거 시 1% 뉘앙스 차이 거부
- **Given**: 시스템 내부에 "N-ONE 아키텍처는 Go 언어로 작성된다"는 지식이 이미 존재하는 상태
- **When**: "N-ONE 아키텍처는 Go 1.22 버전을 권장하며 작성된다"라는 외부 지식이 병합 파이프라인에 도달했을 때
- **Then**: `SemanticComparator`가 코사인 유사도를 0.98로 계산하고, 임계치(1.0) 미만이므로 병합을 거부하며 원본을 개별 지식 노드로 보존한다.

### Scenario 3: 5대 렌즈 추론 누락 시 상태 전이 거부
- **Given**: 지식 수집이 무손실 검증을 통과한 상태
- **When**: 시스템이 IT, 비즈니스, 교육 렌즈까지만 통과하고 에러로 인해 예술, 실생활 렌즈 평가를 건너뛰려 할 때
- **Then**: `Domain Lens Analyzer`의 Bitmask가 `0b11111`을 충족하지 못해 최종 방향성 도출 단계로의 상태 전이를 거부(Error 리턴)한다.

## Edge Cases

### Edge Case 1: 특수 문자로 인한 파싱 데이터 유실
- **Given**: 이모지 및 제어 문자열이 다수 포함된 악의적이거나 복잡한 페이로드 입력
- **When**: `LosslessValidator`가 스트리밍 해싱으로 원본과 추출본을 비교할 때
- **Then**: 해시 불일치가 즉각 감지되며 `panic`을 트리거하고, `.autopus/history/lessons_learned.md` 파일에 "특수문자 파싱 오류로 인한 Byte 손실 감지"를 Append 모드로 명시적 로깅한 뒤 종료된다.

- [ ] 터미널에 5대 방향성 리스트가 리스트 형태로 명시적 출력되는 E2E 동작 확인
- [ ] 10x 리뷰 루프(the_10x_gemini_legion_protocol)를 모두 통과한 최종 커밋
