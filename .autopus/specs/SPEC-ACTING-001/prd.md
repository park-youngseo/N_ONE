# PRD: N-ONE 성인 연기학원 수강생 대시보드 (SPEC-ACTING-001)

## 1. Overview
N-ONE 성인 연기학원 수강생들이 자신의 출결 상태와 대본 연습 진도를 한눈에 확인하고 관리할 수 있는 웹 기반 대시보드를 구축합니다. 수강생에게는 학습 동기 부여를, 관리자에게는 효율적인 학생 관리를 제공하는 것을 목표로 합니다.

## 2. Context & Discovery Q&A
- **Q: 현재 출결은 어떻게 관리되나요?**
  - A: 현재는 수동으로 관리되거나 별도의 시스템이 부재한 상태로 가정합니다. 본 시스템이 통합 관리의 시작점이 됩니다.
- **Q: 대본 연습 진도는 어떻게 정의되나요?**
  - A: 대본의 각 장면(Scene)별 숙지 여부, 감정 분석 완료 여부, 리허설 완료 여부 등으로 세분화됩니다.
- **Q: 주 타겟 사용자는 누구인가요?**
  - A: N-ONE 연기학원에 등록된 성인 수강생 및 학원 운영진입니다.

## 3. Goals & Success Metrics
- **Goals**:
  - 수강생이 자신의 학습 진행 상황을 투명하게 확인.
  - 출결 누락 방지 및 개인별 진도 최적화.
- **Success Metrics**:
  - 수강생 대시보드 접속 빈도 (주 3회 이상 목표).
  - 대본 연습 진도 업데이트 지연 시간 감소.

## 4. Target Audience
- **수강생**: 출결 확인, 대본 다운로드, 연습 진도 체크.
- **강사/관리자**: 수강생별 출결 현황 파악, 진도 피드백 제공 (초기 단계에서는 수강생 뷰 위주).

## 5. Job Stories & User Stories
- **Job Story**: "수업에 참여할 때, 내 출결 상태를 확인하여 수료 조건을 충족하고 있는지 알고 싶다."
- **User Story**: "수강생은 대시보드 메인에서 이번 달 출석률을 그래프로 볼 수 있어야 한다."
- **User Story**: "수강생은 현재 연습 중인 대본의 진행률(%)을 확인하고 완료된 장면을 체크할 수 있어야 한다."

## 6. Functional Requirements
- **출결 관리**:
  - 월별/주별 출석 현황 달력 보기.
  - 지각, 결석, 출석 구분 표시.
- **대본 연습 진도**:
  - 대본 목록 및 상세 보기.
  - 장면별 체크리스트 (읽기 완료, 암기 완료, 감정 분석 완료, 리허설 완료).
  - 전체 진행률 프로그레스 바.
- **개인 프로필**:
  - 수강 중인 클래스 정보 표시.

## 7. Non-Functional Requirements
- **응답성**: 대시보드 로딩 시간 1.5초 이내.
- **모바일 최적화**: 모바일 브라우저에서도 보기 편한 반응형 디자인.
- **보안**: 개인 정보 및 진도 데이터 보호 (JWT 기반 인증).

## 8. UI/UX Concept
- **Visuals**: Modern, Clean, Professional. Acting focus (극장/무대 느낌의 다크 모드 또는 세련된 미니멀 화이트).
- **Interactive Feedback**: 체크박스 선택 시 애니메이션 효과, 진도 상승 시 축하 효과.

## 9. Risk Analysis & Pre-mortem
- **Risk**: 수강생이 진도를 수동으로 업데이트하지 않을 가능성.
  - **Mitigation**: 업데이트 시 보상(배지 등) 시스템 또는 강사 확인 기능 연동.
- **Risk**: 데이터 동기화 지연.
  - **Mitigation**: 낙관적 업데이트(Optimistic UI) 적용.

## 10. Out of Scope
- 실시간 영상 수업 기능 (초기 버전).
- 수강료 결제 시스템 (추후 확장).

## 11. Milestones
- M1: 아키텍처 설계 및 DB 스키마 정의.
- M2: API 개발 (출결, 대본 진도).
- M3: 프론트엔드 대시보드 UI 구현.
- M4: 통합 테스트 및 배포.
