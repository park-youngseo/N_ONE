# SPEC-ACTING-001 Acceptance: N-ONE 성인 연기학원 수강생 대시보드

## Test Scenarios

### Scenario 1: 출결 현황 조회 및 캐시 히트
- **Given**: 출결 데이터(출석 10회, 지각 2회)가 있는 수강생이 로그인한 상태
- **When**: 대시보드 메인 페이지에 접속했을 때
- **Then**: Redis 캐시를 통해 0.5초 이내에 데이터가 로드되며, 이번 달 출석률이 83% (10/12)로 정확히 표시되고 달력에 해당 날짜가 올바른 색상으로 강조된다.

### Scenario 2: 대본 연습 진도 실시간 브로드캐스팅
- **Given**: '로미오와 줄리엣' 대본의 Scene 1을 연습 중인 수강생이 브라우저 탭 A와 탭 B를 동시에 열어둔 상태
- **When**: 브라우저 탭 A에서 Scene 1의 '대사 암기' 체크박스를 클릭하여 진행률을 올렸을 때
- **Then**: 탭 A의 로컬 상태가 즉시 낙관적 업데이트(Optimistic Update) 되고, 탭 B의 진행률 바도 WebSocket 이벤트를 수신하여 새로고침 없이 즉시 상승한다.

### Scenario 3: 다크 모드 전환 (UX 브랜드 반영)
- **Given**: 기본 라이트 모드 상태의 대시보드
- **When**: 테마 설정에서 '다크 모드'를 선택했을 때
- **Then**: 배경이 짙은 회색/검정으로 변경되며, 주요 강조 색상이 연극 무대의 조명 느낌(골드/레드)으로 변경된다.

## Edge Cases

### Edge Case 1: WebSocket 연결 단절 및 복구
- **Given**: 데이터 조회 후 모바일 기기가 터널에 진입하여 인터넷 연결이 끊긴 오프라인 상태
- **When**: 진도 업데이트(대사 암기 체크)를 2회 시도하고, 이후 터널을 빠져나와 인터넷이 복구되었을 때
- **Then**: 오프라인 상태 중의 시도는 로컬 큐에 저장되며 사용자에게 "연결 대기 중" 알림을 띄우고, 인터넷 복구 시 큐에 적재된 2회의 업데이트가 서버로 동기화되어 최종 진도율이 정확히 반영된다.

### Edge Case 2: 데이터가 없는 신규 수강생
- **Given**: 오늘 처음 등록하여 출결 및 대본 데이터가 전혀 없는 수강생
- **When**: 대시보드 접속 시
- **Then**: '데이터가 없습니다. 첫 수업을 기다려주세요!'와 같은 친절한 Empty State 메시지가 표시된다.
