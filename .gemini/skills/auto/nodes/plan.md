# Node: auto plan
## Global Connectivity Table
| Node | Role | Connectivity |
| :--- | :--- | :--- |
| SKILL.md | Root Router | Calls this node for `plan` |
| idea.md | Brainstorming | Provides context via `--from-idea` |
| plan.md | SPEC Planning | Delegates to `planner` and `spec-writer` |
| go.md | Implementation | Implements the SPEC generated by `plan` |

## Description
Write a SPEC document. Delegates to `planner` (PRD) and `spec-writer` (SPEC) agents.

## Flags
| Flag | Description | Default |
| :--- | :--- | :--- |
| `--from-idea` | Create SPEC from a brainstorm file | N/A |
| `--skip-prd` | Skip PRD generation (Medium difficulty) | `false` |
| `--prd-mode` | PRD mode: `standard` (10 sections) or `minimal` (5 sections) | `standard` |

## Pipeline Steps
1. **Parse Flags**: Extract `FEATURE_DESC`, `SKIP_PRD`, `PRD_MODE`.
2. **PRD Generation**: [Conditional] Spawn `planner` agent to write `prd.md`.
3. **SPEC Generation**: [Required] Spawn `spec-writer` agent to write SPEC files.
4. **Review Gate**: [Conditional] Multi-provider review via `auto orchestra review`.

## SPEC File Nodes
| File | Responsibility |
| :--- | :--- |
| `prd.md` | Product Requirements Document |
| `spec.md` | Core Specification (EARS requirements) |
| `scenarios.md` | E2E Verification Scenarios |
| `implementation_plan.md` | Step-by-step developer guide |
