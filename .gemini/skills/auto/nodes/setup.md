# Node: auto setup
## Global Connectivity Table
| Node | Role | Connectivity |
| :--- | :--- | :--- |
| SKILL.md | Root Router | Calls this node for `setup` |
| setup.md | Context Generation | Analyzes codebase via Explorer agent |
| status.md | Dashboard | Relies on docs generated by `setup` |

## Description
Analyze the project's architecture, structure, and tech stack to produce context documents.

## Pipeline Steps
1. **Analyze Codebase**: Spawn `explorer` agent to extract structure, tech stack, and APIs.
2. **Generate ARCHITECTURE.md**: Document domains, layers, and dependency maps.
3. **Generate Project Docs**: Update `.autopus/project/` (product, structure, tech).
4. **Generate scenarios.md**: Extract user-facing E2E test scenarios.
5. **Generate canary.md**: Health check configuration (build, endpoints, browser targets).

## Target Files
| File | Responsibility |
| :--- | :--- |
| `ARCHITECTURE.md` | Core architecture map |
| `.autopus/project/product.md` | Product definition |
| `.autopus/project/structure.md` | Directory/Package map |
| `.autopus/project/tech.md` | Technology stack & patterns |
| `.autopus/project/scenarios.md` | E2E verification scenarios |
| `.autopus/project/canary.md` | Health check config |
