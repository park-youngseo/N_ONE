```
구글이 압도적으로 치고 나갑니다.
단어 하나만 입력해도 나노 바나나로
여러 장의 이미지를 만들어 주고
이곳으로 넘어가 배우 3.1이
영상으로 변환하고 키워드 하나면 여러
장의 고퀄리티 영상이 만들어지는데요.
최근 한국에 도입된 구글의 58인데
나노 바나나부터 배우 3.1을 최대
10번 이상 무료로 제작할 수 있어서
지금 바로 사용해 보셔야 됩니다.
노드를 처음 보면 조금 복잡해 보일
수 있는데 개발 지식이 전혀 없는
일반인들도이 영상 하나로 여러 업무를
자동할 수 있도록 오늘도 핵심만
담았습니다. 먼저 구글 오파를
검색하고 이곳으로 들어와 오른쪽
상단을 클릭하면 구글 로그인이
가능합니다. 이후 아래 갤러리에는
이미 제작된 워크플로우를 참고할 수
있는데 기본 사용법 먼저 알려
드릴게요. 크레인을 클릭하면 바로
제작이 가능한데 사용법은 의외로
간단합니다. 인풋, 제너레이트,
아웃풋 세 가지만 연결해 주면
되는데요. 쉽게 비유를 하자면이
캔버스는 하나의 공장이고 인풋은
제품을 주문하는 사람. 제너레이트는
각 공정의 작업자. 아웃풋은 완성된
제품의 출구인데요. 우선 키워드
하나로 디테일한 프롬프트, 이미지,
영상이 한 번에 만들어지는 과정을
예시로 보여 드릴게요. 첫 번째로
인풋, 제너레이트, 아웃풋을 클릭하면
이런 네모 박스가 생기는데 이런
박스를 노드라고 부릅니다. 중요한 건
하나의 작업이 정상적으로 작동하기
위해서는 인풋, 제너레이트, 아웃풋
노드가 각각 하나 이상은 들어가야
되는데 여기서 핵심은 작업자임
제너레이트 노드입니다. 클릭하면
오른쪽 스텝에 이런 창이 뜨는데요.
이곳은 노드의 이름을 지정하는 곳인데
결과물에는 영향이 없어서 우리가
알아볼 정도로만 표기해 주면 됩니다.
그리고 아래를 클릭하면 현재 구글에서
제공 중인 여러 모델들을 선택할 수
있는데 플래시는 일상적인 작업을
빠르게 처리할 때 효과적이고 프로는
복잡한 문제를 추론할 때 플랜은
단계별 계획을 실행할 때 딥리서치는
긴 시간을 소유해 방대하고 신층적인
연구를 할 때 이마진과 나노 반환한
이미지를 생성할 수 있고 오디오는
음성 생성 배우는 영상 생성
라이리아는 배경음을 생성할 때
사용되는 모델인데 저는 간단한 작업은
플래시 복잡한 작업은 프로를 사용하고
이미지는 최신 팀 모델인 나노바나를
사용하는 걸 추천드립니다. 저희가
만들 작업은 키워드 하나로 프롬프트,
이미지 영상 제작이니 첫 번째
제너레이트 노드는 프롬프트 작성을
위해 2.5 플래시를 지정해 주고
아래 프롬프트에는이 정도로 간단하게
적어 둘게요. 이곳에 들어가는
지침이이 노드의 실제 역할이어서 사용
목적에 맞게 프롬프트를 작성하시면
됩니다. 그리고 모든 노드는 선으로
연결시켜 줘야 되는데요. 공장의
컨베이어 벨트라고 생각하시면 됩니다.
이곳을 클릭한 채 움직이면 선이
나오고 다른 노드에 연결이 가능한데
이런 선을 와이어라 부르고 선이
나오는 연결 지점을 포트라고
부릅니다. 사실 저희들끼린 박스,
선, 점 이렇게 불러도 상관없는데이
명칭들은 기존 제피어나 엠파링 같은
노드 기반 자동화 툴에도 쓰이는 기본
용어들이라 미리 익혀 두시면 다른
모델을 쓰더라도 쉽게 이해할 수 있을
겁니다. 연결을 하고 제너레이트
스텝을 다시 보면 인풋 노드가 연결된
걸 볼 수 있는데요. 정확히
연결됐는지 확인하기 위해서는 왼쪽
프리뷰를 클릭하고 새로 고침을 누르고
스타트를 누르고 키워드를 입력해
테스트해 볼 수 있습니다. 결과물을
보니 요청한 대로 프롬프트를 잘
작성해 줍니다. 이후 이미지를 제작할
노드를 만들어 주면 되는데 참고로
노드는 클릭한 채로 위치를 조절할 수
있고 스페이스바와 마우스를 누른 채로
움직여 캔버스 위치를 조절하거나
컨트롤을 누른 채로 마우스 휠를
돌리면 확대 축소가 가능하고 이곳을
클릭하면 정 사이즈로 화면을 맞춰
주는데 이곳에서 컨트롤을 누르고 휠를
돌리면 웹페이지 사이즈 자체가
변경돼서 캔버스 위에서 사이즈를
조절하셔야 됩니다. 이제 다시 속도를
좀 내서 제너레이트를 클릭해 노드를
생성하고 이름을 설정하고 모델은
이미지 제작 모델인 나노 바나나로
설정해 주고 아래에선 제작할 이미지의
비율을 설정할 수 있는데 저는
16대9로 설정해 둘게요. 그리고
노드를 연결하고 다시 프리뷰에서 새로
거침을 클릭하고 테스트하면 이제
키워드 하나로 이미지까지
만들어지는데요. 이번 영상의 테스트용
키워드는 모두 크리스마스로 통일해서
진행하겠습니다. 다음으로 영상
제작까지 한 번에 해 볼 텐데 노드를
생성하고 영상 제작 모델인 배오를
선택하고 와이어로 연결하고 아래에선
모델을 설정할 수 있는데 과장 채신
모델인 배우 3.1을 사용할 수 있고
비율을 지정하고 아래이 설정은
자동으로 프롬프트를 보정해 주는
기능인데 프롬프트를 간단하게 입력해서
고퀄리티 영상을 만들고 싶은 분들은
그대로 두면 되고 AI 자체 보정
개입을 최소한으로 하고 싶은 분들은
체크해서 사용하시면 됩니다. 이후
테스트를 진행하면 이곳에서 실시간
작업 흐름을 확인하거나 오른쪽 컨솔을
누르면 각각의 노드에서 어떤
결과물들이 나왔는지 확인할 수 있는데
이번엔 오류가 떴습니다. 오류가 뜨면
왼쪽 화면에 빨간색 테두리가 생기고
콘솔을 클릭하면 오류가 난 부분을
자세히 확인할 수 있는데 영상 생성을
위한 프롬프트를 입력하라고 합니다.
스텝을 클릭해 프롬프트를 입력하고
다시 영상을 제작해 보겠습니다.
이번엔 오류 없이 프롬프트부터
영상까지 한 번에 제작됐는데요.
이미지는 다른 지시 없이 제작이
가능하고 영상은 프롬프트가 필수인데
사용하면서 오류가 발생하면 콘솔에서
오류를 확인하고 대처할 수 있습니다.
작업의 마지막으로 아웃풋 노드를
연결해 주면 되는데 이곳에선 결과물
표기 형식을 볼 수 있습니다. 예를
들어 메뉴얼를 클릭하면 이런 식으로
웹페이지를 클릭하면 이런 식으로
만들어지고 그리고 아래 독스 슬라이드
시트 형태로 제작도 가능한데 매뉴얼이
가장 무난하게 사용할 수 있습니다.
우리가 결과물을 받아보는 건 아웃풋
창인데 영상만 표기할 수 있고 각각의
노드를 아웃풋으로 연결하면 모든
값들을 한 번에 볼 수 있는데 컨트롤
C를 눌러 복사하고 컨트롤 V 붙여
넣기를 하면 위치를 조절할 수 있고
딜리트 키를 누르면지를 수 있어서
결과물 구조를 디테일하게 조절할 수
있습니다. 노드를 옮겨 보면 와이어가
정확히 연결되 있는 걸 볼 수 있는데
새로고침을 누르고 중앙 상단 앱을
눌러서 완성된 결과물을 테스트해
보겠습니다. 결과물이 나오는 땐 2분
정도 걸렸고 저희가 설정한 매뉴얼
형식에 인풋과 프롬프트, 이미지,
영상 결과물까지 한 번에 완성된 걸
볼 수 있는데 이곳을 클릭해 각각
다운받을 수 있습니다. 그리고 테마를
누르면 이미지를 설정할 수 있고 왼쪽
상단에서 캠버스 이름을 지정할 수
있어서 여러 캔버스를 홈탭에서 쉽게
구분할 수 있습니다. 사실 오파를
처음 쓰는 분들에게는 난이도가 있는
기술이라 최대한 쉽게 설명하라고
테스트만 이틀 넘게 했는데 영상의
좋아요와 구독 한 번씩만
부탁드릴게요. 다음으로 좀 복잡한
작업을 만들어 볼 텐데 기본적인 작동
방식만 이해하셨다면 크게 어렵진
않습니다. 이번엔 크롬포트 하나로
배경음, 나레이션 그리고네 개의
영상을 한 번에 만들어 볼 건데
이해를 돕기 위해서 기존 제작한
워크플로우로 업그레이드 해
보겠습니다. 먼저 기존 와이어를
클릭해 제거해 주고 좀 더 복잡한
작업인만큼 두 번째 노드 모델은
프로로 변경해 주겠습니다. 그리고
제작에 필요한 프롬프트를 요청해 볼
건데 저는 이렇게 입력해 보겠습니다.
제가 입력한 프롬프트는 모두 예시라
가볍게 참고하시고 상황에 맞게
수정해서 사용하시면 됩니다. 이후
각각의 이미지 제작 프롬프트를 따로
분리하고 뒤에는 나노바나 노드를
연결해 주겠습니다. 이렇게 하는
이유는 하나의 노드에서 여러
프롬프트를 생성하고 한 번에 와이어를
연결하면 간혹 이미지가 여러 장
만들어지는 오류가 발생해서 프롬프트를
각각 분리하는 과정을 따로
추가했습니다. 결과물을 테스트해 보니
각각의 이미지가 잘 만들어지는 걸 볼
수 있습니다. 그리고 나레이션
프롬프트 제작을 위한 노드를 생성하고
모델을 설정하고 프롬프트를 입력하고
바로 뒤에 나레이션 제작 노드를
설정하고 오디오 모델을 연결했는데
아래를 클릭하면 제작 음성의 성별을
설정할 수 있습니다. 참고로 오디오
노드에 입력되는 프롬프트는 모두
음성으로 변환돼서 대사만 필터링 할
수 있는 노드를 따로 제작하는게
중요합니다. 이후 배경 제작 프롬프트
노드를 생성하고 바로 뒤에 배경음
제작 노드를 생성해 모델을 설정하고
와이어로 연결해 주시면 됩니다.
지금까지 작업한 내용을 테스트해 보니
내장의 이미지와 배경은
나레이션까지 한 번에 제작되는 걸 볼
수 있는데요.
```
