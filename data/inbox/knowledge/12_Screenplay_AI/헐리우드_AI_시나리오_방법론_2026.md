## [섹션 1: 2026년 헐리우드 AI 시나리오 트렌드 — 핵심 요약]

### 패러다임 전환

> **"AI가 대본을 다 써주길 바라는 사람"이 아니라, "AI를 능숙하게 부려 본인의 창의적 비전을 더 빠르게 구현하는 사람"이 살아남는다.**

| 구분 | 과거 (2023 이전) | 현재 (2026) |
|------|----------------|------------|
| AI 역할 | 텍스트 생성기 | 창의적 파트너(Co-Writer) + 개발 인프라 |
| 초안 작업 기간 | 2~3주 | 며칠 단위 |
| 협업 방식 | 개인 작업 | 실시간 AI 내장 협업 도구 |
| 출력물 | 텍스트 스크립트 | 텍스트 + 스토리보드 + 시각화 영상 |

### 2023 헐리우드 작가 파업 이후 확립된 가이드라인

1. **AI는 '작가'로 인정되지 않음** — 저작권은 인간 작가에게만 귀속
2. **AI 사용 강요 금지** — 제작사는 작가에게 AI 사용을 강제 불가
3. **투명성 의무** — AI 활용 자료가 포함된 경우 반드시 고지
4. **크레딧 보호** — AI 초안을 수정했더라도 작가는 정당한 크레딧과 보수를 받음

## [섹션 2: 2026년 표준 AI 시나리오 워크플로우 — 5단계]

### 1단계: 구조 설계 및 비트 시트 생성

- **입력**: 아이디어 → 로그라인 작성
- **AI 역할**: 3막 구조에 맞는 비트 시트(Beat Sheet) 자동 제안
- **프레임워크 선택**: Save the Cat / Hero's Journey / Truby's 22 Steps
- **주의**: AI가 생성한 템플릿을 그대로 쓰지 말고, 반드시 수동으로 비틀어라 → 진부한 결과 방지

### 2단계: 스토리 바이블(Story Bible) 구축

- **입력**: 캐릭터 배경, 동기, 말투(Style), 세계관 규칙
- **AI 역할**: 작품 전체에서 캐릭터 일관성 유지 — "캐릭터 드리프트" 방지
- **도구**: Sudowrite Story Bible 기능 / Claude(대용량 컨텍스트 창)
- **산출물**: 작품 전체를 관통하는 캐릭터 아크, 세계관 규칙, 서사 스레드 문서

### 3단계: 장면 집필 및 확장 (Scene Writing)

- **입력**: 작가가 핵심 감정 흐름과 장면 방향을 입력
- **AI 역할**: 적절한 묘사와 대사를 생성 → 작가가 수정하여 '맛' 살리기
- **핵심 기법**: 반복 프롬프팅(Iterative Prompting) — 한 번에 완성 기대 금지
  1. 전제(Premise) 입력
  2. 아웃라인 요청
  3. 아웃라인 비판
  4. 톤 조정 요청
  5. 반복 정제

### 4단계: 분석 및 수정 (Structural Review)

- **AI 역할**:
  - 플롯 홀(Plot Hole) 탐지
  - 페이싱(Pacing) 문제 분석
  - 수정 제안 제공
- **스트레스 테스트 질문 예시**:
  - "이 이야기가 끝날 수 있는 3가지 방식은?"
  - "이 캐릭터가 이 결정을 내리는 이유는?"
  - "이 장면을 50% 짧게 만들면 무엇이 남는가?"

### 5단계: 오디오 피드백 & 시각화

- **오디오**: AI 음성 생성 도구로 대사를 실제 목소리처럼 들어보며 자연스러움 최종 점검
- **시각화**: LTX Studio — 스크립트 입력 → 즉시 스토리보드 + 영상 콘티 생성
  - 제작진과 비주얼 공유 (피치 덱용)
  - 인물·장소 시각적 일관성 유지 ("Elements" 잠금 기능)

## [섹션 4: NONE_시나리오 프로젝트에 즉시 적용 가능한 방법]

### 현재 프로젝트 상황
- **장르**: 누아르 범죄 (Sicario 스타일)
- **구조**: 블록 시스템 (현재 블록 05 진행 중)
- **캐릭터**: 조(Joe), 니코(Niko), 최준혁, 박왕열 등

### 즉시 적용 가능한 AI 기법

| # | 방법 | 구체적 적용 |
|---|------|-----------|
| 1 | **스트레스 테스트** | Claude에 MASTER_SHEET + 블록05 초안 업로드 → "이 블록에서 플롯 홀을 찾아라" 명령 |
| 2 | **대사 최적화** | Sudowrite에 조의 대사 입력 → "이 인물의 말투를 더 냉정하고 간결하게" 요청 |
| 3 | **비트 시트 검증** | ChatGPT에 현재 블록 구조 입력 → "Save the Cat 기준으로 누락된 비트를 찾아라" |
| 4 | **시각화 피치** | LTX Studio에 블록05 씬을 입력 → 스토리보드 자동 생성 |
| 5 | **캐릭터 일관성** | Claude에 전체 캐릭터 바이블 입력 → 새 장면에서 캐릭터 말투/행동 자동 검증 |

### NONE 프로젝트에 Claude를 사용하는 이유

> Claude는 대용량 컨텍스트 창(200K 토큰 이상)을 지원하여, MASTER_SHEET + 전체 블록 초안 + 캐릭터 바이블을 **동시에 메모리에 유지**하면서 작업할 수 있다. 복잡한 다층 서사 구조에서 캐릭터 일관성을 유지하는 데 가장 적합하다.

## [섹션 5: 핵심 원칙 — AI 시나리오 작업의 불문율]

1. **AI는 비계(Scaffolding)다** — 구조를 잡는 데 쓰고, 창의적 비전은 반드시 작가 본인이 결정
2. **한 번의 프롬프트로 완성 기대 금지** — 반복 프롬프팅(Iterative Prompting)이 표준
3. **스토리 바이블 선행** — 캐릭터 바이블 없이 장면 집필하면 일관성 붕괴
4. **AI 제안은 반드시 비틀기** — AI 출력을 그대로 쓰면 진부한 결과 → 반드시 수동 변형
5. **목소리 보존** — AI 제안 후 본인 말로 재작성하는 것이 핵심 기술
6. **플롯 홀 선제 탐지** — 장면 집필 전 구조를 AI로 먼저 분석하면 나중에 대규모 수정 방지
