> 요약 또는 압축 없이 원자 단위로 전체 내용을 보존합니다.

```
달라고 했는데 우선 리스트를 받을 때
동안은 테스트 스프라이트가 작동을 안
하는게 좋습니다. 그래서 저렇게
프롬프트를 입력을 해 준 거고요.
리스트를 쭉 복사를 합니다. 자,
복사를 한 다음에 메모장을 열어서이
리스트를 붙여 넣기를 해 주시고요.
바탕 화면이나 편하신 폴더에 저장을
해 놓습니다. 그다음에 이제 테스트
스프라이트를 실행해서이 프로젝트를
테스트해 달라는 프롬프트를 입력을
하는 겁니다. 자, 그러면 이렇게
테스트 셋업이 나오거든요. 저는
디자인을 하고 싶기 때문에 모드는
프런트 엔드로 고정을 해 줄 거고요.
만약에 여러분이 테스트하고 싶은게
로직이다. 백엔드적인 부분이다.
그러면 백엔드 탭을 선택을 해 주시면
되겠습니다. 저는 프런트 엔드로 할
거고 전체 코드를 테스트하고 싶기
때문에 코드 베이스로 해 줬고요. 쭉
내려가서 현재 개발 서버가 포트
3,000에 되어 있기 때문에 건들지
않도록 하고요 마지막에 프로덕트
스페시피케이션 닭이 있습니다. 바로
이곳에 저희가 만들어 놨던 텍스트
파일을 업로드를 해 주신 다음에
컨티뉴 버튼을 눌러 주시면 됩니다.
업로드가 석세스 됐고요. 확인을
눌러서 닫아 주시면네 안티
그래비티에서이 테스트 스프라이트
서버가 작동을 하고 있습니다. 그래서
뭐 필요한 코드를 생성을 해 준다거나
혹은 필요한 테스트를 써 준다거나
이런 거를 다 자동으로 실행을 할
거거든요. 여기 왼쪽에 테스트
스프라이트 테스트라는 폴더가 새로
만들어진 걸 확인할 수가 있죠.
그리고 우리 프로젝트에 대해서
전체적인 코드 서머리까지 만들어
줍니다. 이걸 베이스로 이제 디버깅을
시작을 하는 거거든요. 자, 지금
우측에서 테스트가 진행이 되고
있다라고 하죠. 여러분이 테스트
스프라이트 웹사이트를 돌아가시면
실시간으로 해당 테스트가 진행이 되는
걸 확인을 할 수가 있습니다. 이게
전체 테스트가 작동하는데 시간이 좀
걸리거든요. 그래서 여러분이 다른 거
하시거나 그래서이 테스트가 진행되는
시간 동안 여러분이 다른 일을
하시다가 테스트가 다 끝나고
돌아오셔서 작업을 이어가시는 걸
추천드립니다. 테스트가 다 끝나고
나면 여러분이 직접 하나하나 눌러서
어떤 식의 테스트가 진행이 됐고 어떤
식의 결과물이 나왔는지 이렇게
영상으로 확인할 수도 있습니다. 지금
제가 실패했다고 나오는 건이
레이아웃이랑 호버 이펙트인데 아무래도
로딩이 너무 빨리 돼서 안 된게
아닌가 싶거든요. 한번 고쳐 달라고
해 보도록 하겠습니다. 그리고 어떤
식의 테스트가 진행이 됐고 실패했다면
왜 실패했는지 이런 모든 분석이 적혀
있는 파일도 받아볼 수가 있습니다.
그래서 지금이 실패한 테스트 있죠?
이거를 그대로 가져다가이 부분을
해결해 달라고 프롬프트를 입력을 해
줬고요. 결과물을 확인해 보면 이렇게
호버 이펙트가 제대로 작동을 한다던가
제품을 카트에 넣었을 때이 카트에도
이미지가 정상적으로 표시되는 등 여러
가지 버그나 디테일이 업그레이드 된
걸 확인할 수가 있었습니다. 제가
디자인적인 부분이 가장 직관적이라
디자인으로 보여 드렸지만 여러분이이
테스트 스프라이트를 사용해서 결과물의
퀄리티를 올리고 싶으실 때는 백엔드
테스팅이 훨씬 더 강력할 겁니다.
그래서 한 달 동안 무료로 사용해 볼
수 있으니까 꼭 사용해 보시길
바랍니다. 네, 오늘 이렇게 총 세
가지의 MCP 서버를 활용해서 조금
더 기존의 웹사이트보다 훨씬 더
고퀄리티의 웹사이트들을 만들어
봤는데요. 지금까지 내가 어떻게
하면은 좋은 웹사이트를 만들 수
있을까 고민을 하셨던 분들께 조금
도움이 되는 영상이었으면 좋겠습니다.
제 오늘 영상은 여기까지고요. 영상이
도움이 되셨다면 좋아요와 구독,
그리고 알림 설정까지 부탁드립니다.
지금까지 성공 지식과였습니다.
감사합니다.
```
